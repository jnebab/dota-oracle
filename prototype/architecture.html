System Architecture · v1 · draft

DRAFT ORACLE

A Dota 2 draft & live-match advisor. Recommends carries from your pool for your role, weighted by 7.41c meta + bracket, with lineup-tuned item builds — and pulls a live game straight off STRATZ by name or SteamID.

web · React 18 + TypeScript + Vite Tailwind TanStack Query api · FastAPI · Python 3.12 Pydantic v2 · httpx SQLModel + SQLite/Postgres ARQ + Redis pkg · shared TS engine + data (zod) pnpm + Turborepo · uv Docker · Vercel + Fly.io

Key split: the scoring engine + seed data ship inside the client → recommendations are instant and work offline. The backend only does what the browser can't: hold the STRATZ token, call rate-limited external APIs, and refresh meta on a schedule.
02

Live-match track · request flow

1
User → Web
Enters a username or SteamID64 in the Live Import panel.
2
Web → API
GET /api/live/{handle} via TanStack Query.
3
API · steamid service
SteamID64 → 32-bit account id id − 76561197960265728; or a name → OpenDota /search → top match.
4
API → STRATZ
GraphQL player(steamAccountId).liveMatch with Bearer token.
5
API → Web
If live: returns { radiant[], dire[], positions, gameTime }. If not: 404 + last match fallback.
6
Web · engine
Maps heroes onto the enemy/ally board → re-scores your pool → updated recs + item plan, no typing.
